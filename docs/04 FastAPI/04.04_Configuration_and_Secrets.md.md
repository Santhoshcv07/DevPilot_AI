# 04.04 Configuration & Secrets Management

> **The Mission:** Securely load sensitive application configurations (passwords, API keys, environments) without hardcoding them into the version control history.

## 1. Mental Model & Architecture
* **Environment Variables:** Values set outside the application code. Like giving the application a locked briefcase of instructions right as it boots up.
* **The `.env` File:** The actual briefcase on our local machine containing real passwords. **This must always be in `.gitignore`.**
* **The `.env.example` File:** A dummy briefcase committed to Git. It shows other developers the *names* of the keys they need, but leaves the values blank.

**Current Architecture State:**
```text
DevPilot-AI/
├── backend/
│   ├── api/          
│   ├── core/         
│   │   └── config.py # Uses pydantic-settings to read the .env file
│   ├── schemas/      
│   └── main.py       
├── .env              # Real secrets (IGNORED BY GIT)
├── .env.example      # Template secrets (TRACKED BY GIT)
└── .gitignore        # Keeps .env out of version control
