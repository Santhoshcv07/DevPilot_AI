# 04.03 Data Validation & Pydantic

> **The Mission:** Secure the API by ensuring all incoming data strictly adheres to predefined rules before it reaches our backend logic.

## 1. Mental Model & Architecture
* **GET vs. POST:** * `GET` requests ask for data (like viewing a menu). They have no body.
  * `POST` requests send data to the server (like handing in an order form). They contain a JSON body.
* **Pydantic (The Customs Agent):** A Python library that enforces data types and validation rules at the boundary of our application. If data doesn't match the schema, it is rejected immediately.
* **Schema (The Rulebook):** A blueprint defining exactly what a valid JSON payload must look like.

## 2. Core Concepts
* **HTTP 422 Unprocessable Entity:** The standard status code returned when the server understands the data format (JSON) but rejects the contents because it violates validation rules (e.g., a string is too short).
* **BaseModel:** The core Pydantic class we inherit from to define our schemas.
* **Field:** A Pydantic function used to add constraints (like `min_length`, `max_length`) and metadata to individual schema properties.
