# 12.08 Text Chunking & The Sliding Window

> **The Mission:** Prevent LLM Context Window overflow by algorithmically chopping massive documents into smaller, semantic fragments.

## 1. Mental Model & Architecture
* **The Context Window Limit:** LLMs have a strict limit on how much text they can process at once. We cannot send an entire 50-page PDF to the AI in a single prompt.
* **The Meat Grinder:** A backend service that ingests a massive string of text and slices it into an array of smaller strings (e.g., 500 characters each).

## 2. Core Concepts
* **Boundary Cuts:** A naive chunking algorithm might slice a sentence exactly in half, destroying its meaning.
* **Chunk Overlap (The Sliding Window):** To prevent destroying context at the boundaries, we force chunks to overlap. If Chunk 1 ends at character 500, Chunk 2 doesn't start at 501. It steps backward (e.g., to character 450) and starts there. This ensures that any sentence cut off at the end of Chunk 1 is repeated in full at the beginning of Chunk 2.

## 3. Implementation Snapshot
**File:** `backend/services/chunking.py`
```python
def get_text_chunks(text: str, chunk_size: int = 500, chunk_overlap: int = 50):
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks.append(text[start:end])
        start += (chunk_size - chunk_overlap) # The sliding window steps back
    return chunks
