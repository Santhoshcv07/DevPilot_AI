# 12.03 External AI Services & The Chef Pattern

> **The Mission:** Connect the application to an external AI inference engine (Groq) while keeping the complex integration logic completely isolated from the API routing layer.

## 1. Mental Model & Architecture
* **The Services Layer (The Chef):** A dedicated folder (`services/`) where complex business logic, third-party API calls, and heavy processing live. The API Routers (Waiters) should never contain this logic; they simply hand the input to the Service and wait for the result.
* **Groq API & LLaMA:** Groq is an inference engine powered by specialized hardware (LPUs) designed to run Open Source models (like Meta's LLaMA 3.1) at incredibly high speeds.

## 2. Core Concepts
* **Model Decommissioning:** AI providers frequently update their infrastructure. Hardcoded model strings (like `llama3-8b-8192`) will eventually be deprecated and disabled, resulting in `400 Bad Request` errors. Applications must be designed to easily swap model strings.
* **System Prompt:** The invisible instructions provided to the AI before the user's prompt (e.g., "You are an elite coding assistant"). It strictly dictates the AI's persona, boundaries, and formatting rules.

## 3. Implementation Snapshot

from groq import Groq
fromcore.config import settings

client = Groq(api_key=settings.groq_api_key)

def get_ai_response(prompt: str) -> str:
    chat_completion = client.chat.completions.create(
        messages=[
            {"role": "system", "content": "You are DevPilot AI..."},
            {"role": "user", "content": prompt}
        ],
        model="llama-3.1-8b-instant", 
    )
    return chat_completion.choices[0].message.content
