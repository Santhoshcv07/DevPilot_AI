# 12.05 Stateful AI Memory & Context Arrays

> **The Mission:** Cure the LLM's amnesia by storing transient chat messages in a persistent relational database, and dynamically rebuilding the conversation history on every API request.

## 1. Mental Model & Architecture
* **The Stateless AI:** AI models (like LLaMA or GPT-4) have zero memory. Every API request is treated as a completely isolated event. 
* **The Stateful Database:** To create a continuous conversation, the backend must act as the AI's long-term memory. It stores every user prompt and AI response in a PostgreSQL `messages` table linked to a specific `chat_id`.
* **The Context Array:** When a user asks a new question, the backend queries the database for all previous messages in that chat, formats them into a chronological list (array) of dictionaries (`[{"role": "user", "content": "..."}, {"role": "assistant", "content": "..."}]`), appends the new question, and sends the massive package to the AI.

## 2. Core Concepts
* **One-to-Many Relationships:** A `User` has many `Chats`. A `Chat` has many `Messages`. This strict hierarchy keeps data organized and allows us to easily fetch the history of a specific session.
* **On-Delete Cascade:** A database rule ensuring that if a parent record (a `Chat`) is deleted, all of its child records (the `Messages`) are automatically destroyed, preventing orphaned data from bloating the server.
