# 12.07 File Uploads & Data Ingestion

> **The Mission:** Build a secure pipeline that allows users to upload physical files from their computers to our server, extract the raw text, and track the file in a relational database.

## 1. Mental Model & Architecture
* **Multipart Form Data:** Web APIs usually send data as JSON (lightweight text). But to send a physical file (like a PDF or image), the browser must use `multipart/form-data`, which breaks the file into a stream of binary bytes.
* **The Ingestion Pipeline:**
  1. The API route accepts the `UploadFile`.
  2. The backend reads the raw bytes.
  3. The backend decodes the bytes into a UTF-8 string.
  4. The backend creates a metadata row in the `documents` SQL table to track who uploaded it.

## 2. Core Concepts
* **RAG (Retrieval-Augmented Generation):** The overarching strategy of giving an AI external knowledge. Before we can retrieve knowledge, we must ingest it.
* **MIME Types Validation:** Never trust an uploaded file. If you expect a text file, verify that `file.content_type` is actually `"text/plain"`. This prevents users from accidentally (or maliciously) uploading executable `.exe` files or corrupted binaries.
