# 12.04 API Orchestration (The Full Pipeline)

> **The Mission:** Connect all isolated backend systems (Security, Validation, Routing, and External Services) into a single, seamless pipeline.

## 1. Mental Model & Architecture
When a secure request hits a fully orchestrated API endpoint, it passes through four distinct layers:
1. **The Bouncer (Security Dependency):** Intercepts the request, reads the HTTP Header, verifies the JWT Signature, and retrieves the User from the database.
2. **The Customs Agent (Pydantic Schema):** Validates the JSON payload to ensure data integrity and type safety.
3. **The Waiter (FastAPI Router):** The actual endpoint function. It takes the validated user and data and passes them to the business logic.
4. **The Chef (Services Layer):** Executes the heavy lifting (like calling the Groq AI API) and returns the final result.

## 2. Core Concepts
* **HTTP Headers:** Invisible metadata sent with web requests. Security tokens (JWTs) are passed here using the format `Authorization: Bearer <token>` to keep them out of the visible URL or JSON body.
* **Dependency Injection (`Depends`):** FastAPI's superpower. By adding `Depends(get_current_user)` to an endpoint, we guarantee that the route *cannot* execute unless the security checks pass first.
