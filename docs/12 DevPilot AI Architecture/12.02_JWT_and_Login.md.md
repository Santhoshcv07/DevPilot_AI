# 12.02 JWTs & Authentication Routing

> **The Mission:** Allow users to securely register and log in to the application, issuing them a cryptographic token (JWT) so they don't have to send their password on every subsequent request.

## 1. Mental Model & Architecture
* **Models vs. Schemas:** * `Models` define the database tables (e.g., storing the `hashed_password`). 
  * `Schemas` define the API rules (e.g., requiring an 8-character `password` on input, but stripping it out on the API response).
* **JWT (JSON Web Token):** Like a hotel keycard. It proves the user has successfully logged in and grants them access to specific resources for a limited time.
* **The Secret Key:** A highly secure string living in the `.env` file used to cryptographically sign the JWT. If a hacker modifies the token, the signature breaks and the server rejects it.

## 2. Core Concepts
* **OAuth2PasswordRequestForm:** A standard FastAPI utility that expects login data to arrive as HTML Form Data (requiring the `python-multipart` library) rather than JSON. It expects fields strictly named `username` and `password`.
* **Token Expiration:** JWTs must expire (e.g., after 60 minutes) to limit the damage if a token is stolen.
* **HTTP 401 Unauthorized:** The standard status code returned when a user provides an incorrect email or password, or attempts to access a protected route without a valid token.

@router.post("/login")
def login(form_data: OAuth2PasswordRequestForm = Depends(), db: Session = Depends(get_db)):
    user = db.query(User).filter(User.email == form_data.username).first()
    
    if not user or not verify_password(form_data.password, user.hashed_password):
        raise HTTPException(status_code=401, detail="Incorrect email or password")

    access_token = create_access_token(data={"sub": user.email})
    return {"access_token": access_token, "token_type": "bearer"}
