# 12.09 Vector Embeddings & JSON Storage

> **The Mission:** Translate English text into mathematical arrays to enable Semantic Search, and store those arrays in the database.

## 1. Mental Model & Architecture
* **Semantic Search vs. Keyword Search:** Standard databases use `Ctrl+F` keyword matching. If you search for "color", it won't find the word "hue". Semantic search solves this by translating words into math. Concepts that mean similar things end up physically close to each other in a multi-dimensional graph.
* **The Embedding Chef:** We use an open-source Machine Learning model (`sentence-transformers/all-MiniLM-L6-v2`) to read a chunk of text and output a 384-dimensional vector (an array of 384 floating-point numbers).

## 2. Core Concepts
* **Infrastructure Pivoting:** Native PostgreSQL does not understand high-dimensional vectors without a C++ extension called `pgvector`. On systems where C++ build tools are unavailable (like standard Windows environments), we must pivot.
* **The JSON Pivot:** Because a vector is ultimately just a standard list of numbers (`[0.12, -0.04, 0.99...]`), we can bypass `pgvector` entirely and store the math in a standard `JSON` database column. We then handle the mathematical search calculations ourselves in Python.
