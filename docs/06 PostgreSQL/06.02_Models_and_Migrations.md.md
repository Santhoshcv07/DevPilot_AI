# 06.02 Database Models & Alembic Migrations

> **The Mission:** Define database tables using Python classes (Models) and securely push those definitions to the PostgreSQL database using version-controlled migrations (Alembic).

## 1. Mental Model & Architecture
* **Database Model:** A Python class that inherits from the SQLAlchemy `Base`. It acts as the blueprint for a specific database table (e.g., declaring column names and data types).
* **Alembic:** "Git for your database." It reads your Python Models, compares them to the current state of your PostgreSQL database, and generates the raw SQL required to make them match.
* **`alembic_version` Table:** An automatically generated table in Postgres that tracks which migration was last applied, preventing duplicate executions.

## 2. Core Concepts
* **Primary Key (`primary_key=True`):** A unique identifier for every row in a table (usually an `id`).
* **Unique Constraint (`unique=True`):** Ensures no two rows can have the same value for this column (essential for emails).
* **Index (`index=True`):** Creates a hidden lookup table for a column, making database searches significantly faster.
* **Server Default (`server_default=func.now()`):** Tells the database to automatically generate a value (like the current timestamp) if the backend doesn't provide one.

## 3. Implementation Snapshot
**File:** `backend/models/user.py`
```python
from sqlalchemy import Column, Integer, String, DateTime
from sqlalchemy.sql import func
from core.database import Base

class User(Base):
    __tablename__ = "users"

    id = Column(Integer, primary_key=True, index=True)
    email = Column(String, unique=True, index=True, nullable=False)
    hashed_password = Column(String, nullable=False)
    created_at = Column(DateTime(timezone=True), server_default=func.now())
